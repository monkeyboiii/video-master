# media/ — heavy files, outside git

Everything under `media/` except this README and `.gitkeep` files is **git-ignored**.
The manifest (`series/.../manifest.yml`) records every file's name, purpose, and
external location — if a file isn't in a manifest, it doesn't exist.

| Directory | Contents | Named per |
|-----------|----------|-----------|
| `raw/` | Camera originals, organized `raw/<VIDEO_ID>/` | shot pattern |
| `selected/` | Trimmed selects (optional — selects normally live as in/out timecodes in the manifest, not files) | n/a |
| `proxies/` | Edit proxies (usually generated by Kdenlive itself) | Kdenlive-managed |
| `overlays/` | Remotion renders, ProRes 4444 `.mov` | overlay pattern |
| `voiceover/` | VO recordings per locale | VO pattern |
| `audio/` | Music and SFX used by episodes | free, but referenced in manifests |
| `timelines/` | Working `.kdenlive` projects | timeline pattern |
| `exports/` | Review/final renders + cover stills | export/cover patterns |

Patterns: `docs/naming-conventions.md`. Music/SFX in `audio/` must be recorded in the
manifest's `assets.audio` with `source` and `license` — only tracks licensed for
commercial social-media use.

## External storage

Every raw/VO/export file gets an `external_uri` in the manifest so it can be found after
this machine's `media/` is cleaned. Canonical layout (mirror of `media/`):

```text
<STORE>/raw/<VIDEO_ID>/...
<STORE>/voiceover/<VIDEO_ID>/...
<STORE>/exports/<VIDEO_ID>/...
```

> `external_uri` from then on. Until then the validator only warns about missing
